hum.lite_sample_sounds.tag_wf_gen(tag_wfgen_map: ~typing.Mapping[~hum.lite_sample_sounds.Tag, ~typing.Callable[[...], ~typing.Sequence]] = (('random', <function random_samples>), ('pure_tone', <function pure_tone>), ('triangular_tone', <function triangular_tone>), ('square_tone', <function square_tone>)), tag_sequence: ~typing.Iterable[~hum.lite_sample_sounds.Tag] | None = None)[source]

Generate tagged waveforms – i.e. (tag, wf) pairs.

Parameters:

  • tag_wfgen_map – A {tag: wfgen, ...} map where wfgen is a callable taking no arguments and returning a sequence

  • tag_sequence – A sequence of tags (that should all be keys of tag_wfgen_map)

Returns:

Make a generator (here with default specs):

>>> gen = tag_wf_gen()

From it, source (tag, wf) pairs (by default, indefinitely).

>>> tag, wf = next(gen)

Since by default, we’re given random tag and wf, the only thing we can verify here is that tag is a string and wf has the default size of 43008.

>>> type(tag)
<class 'str'>
>>> len(wf)
43008

By default the tags are randomly chosen from the tag_wfgen_map, but you can also specify a iterable of tags explicitly. Obviously, the elements of this sequence must actually be keys of tag_wfgen_map – this will be asserted if the iterable is a Sequence.

You can specify a sequence of tags:

>>> gen = tag_wf_gen(
...     tag_sequence=['random', 'pure_tone', 'triangular_tone', 'square_tone']
... )
>>> tag_and_wf_seq = list(gen)  # consume the whole gen
>>> [x[0] for x in tag_and_wf_seq]
['random', 'pure_tone', 'triangular_tone', 'square_tone']

The first 5 numbers of the triangular tone

>>> assert list(tag_and_wf_seq[2][1][:5]) == [-30000, -29900, -29800, -29700, -29600]
